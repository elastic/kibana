# Localization (i18n)

> **Project-specific.** Contract for this repo.

## What to localize

Every **new or changed** string that is **user-visible** or **read by assistive technology** (`aria-label`, `tableCaption`, tooltip `content`, labels, titles, body copy, etc.) must use **`i18n.translate`** — **not** raw literals.

**Do not** run **`name`** on radios, internal ids, or other programmatic-only tokens through i18n.

## Default: `i18n.translate` (`@kbn/i18n`)

```tsx
i18n.translate('messageId', { defaultMessage: 'English fallback' })
```

If the file already uses a shared object (e.g. `i18nTexts.modalTitle`), follow that for new strings. Add an **`i18n`** import only when you add a new **`i18n.translate`** call; keep imports merged at the top.

**Message ids:** reuse an existing id for the same message; otherwise `fileOrComponent.attribute` (e.g. `myTable.tableCaption`); one id ↔ one `defaultMessage` per file; keep `defaultMessage` aligned with nearby UI.

```tsx
<EuiButtonIcon
  aria-label={i18n.translate('myButton.ariaLabel', {
    defaultMessage: 'Refresh results',
  })}
/>
```
