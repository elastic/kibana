APM-focused alert investigation. Given an APM alert (latency, error count, failed transaction rate, or anomaly), correlates service health, traces, exceptions, logs, runtime metrics, and dependencies, and renders inline an APM metrics comparison card, a metric-over-time chart, and a service map for blast radius. Activates when the user asks to investigate, triage, diagnose, or assess the impact of an APM alert, or when an alert attachment is present and its rule type is apm.transaction_duration, apm.error_rate, apm.transaction_error_rate, or apm.anomaly. Do NOT use for non-APM alerts (SLO, infrastructure threshold, log threshold, custom threshold, uptime, or security).
