# @kbn/securitysolution-mitre-catalog

Canonical Kibana MITRE ATT&CK catalog, shared across security-domain packages
and plugins.

This is the **slim** view of the catalog: `id`, `name`, `reference`, `value`,
and (for techniques) `tactics`. No i18n labels, no rendering helpers, no
plugin-specific concerns. Consumers that need translated labels for UI
rendering should layer them on top in their own package.

## Why this package exists

Historically the catalog lived in two places: the SIEM-side autogenerated
`mitre_tactics_techniques.ts` (with inlined `i18n.translate(...)` calls) in
the `security_solution` plugin, plus a hand-stripped JSON snapshot vendored
into `threat_intelligence` to avoid pulling `@kbn/security-solution-plugin`
into its type-check graph. Both consumers wanted the same data; only the
SIEM detection-rule UI needed the i18n labels, and duplicating the catalog
risked drift on every ATT&CK release.

Hoisting the slim data into this shared package removes the dup without
forcing either consumer to depend on the other. The labeled SIEM-side
catalog is now a thin extension on top of these slim types, written by the
same `yarn extract-mitre-attacks` invocation that updates the package's
JSON, so the two cannot drift.

## Public API

```ts
import {
  // Types
  type MitreTactic,
  type MitreTechnique,
  type MitreSubTechnique,
  type MitreAttackCatalog,

  // Data
  tactics,
  techniques,
  subtechniques,
  CATALOG_META,

  // Lookups
  techniqueById,
  subtechniqueById,
  tacticsToIds,
} from '@kbn/securitysolution-mitre-catalog';
```

`tacticsToIds` translates technique `tactics` (camelCase tactic `value`s as
used in the catalog) back to canonical `TAxxxx` IDs — useful for
detection-rule handoff and ATT&CK heatmap rendering.

## Regenerating

The catalog at `src/data/mitre_attack_catalog.json` is rewritten by the
canonical security_solution extractor on every MITRE ATT&CK release bump:

```
cd x-pack/solutions/security/plugins/security_solution
yarn extract-mitre-attacks
```

That single command updates both this package's slim JSON and the labeled
TS catalog at
`x-pack/solutions/security/plugins/security_solution/common/detection_engine/mitre/mitre_tactics_techniques.ts`,
so the two cannot drift. Bump `MITRE_CONTENT_VERSION` in the extractor
(`scripts/extract_tactics_techniques_mitre.js`) when you adopt a new
ATT&CK release; the version must match what the prebuilt rules
package at `https://github.com/elastic/detection-rules` is using.
