# Supported links usage with examples:

## Uptime

To use the link `nodeType` and `node` should be provided. The `nodeType` is set to `host` for the purpose of the context.

```tsx
const node = {
    name: 'host1',
    id: 'host1-macOS',
    title: {
    name: 'host1',
    cloudProvider: null,
    },
    os: 'macOS',
    ip: '192.168.0.1',
    rx: 123179.18222222221,
    tx: 123030.54555555557,
    memory: 0.9044444444444445,
    cpu: 0.3979674157303371,
    diskLatency: 0.15291777273162221,
    memoryTotal: 34359738368,
}
<LinkToUptime nodeType="host" node={node} />
```

<Story id="infra-host-details-view-links--uptime-link" />

The link should look similar to

`/base-path/app/uptime/?search=host.name: "host1" OR host.ip: "192.168.0.1" OR monitor.ip: "192.168.0.1"`

## APM Services

To use the link `hostname` and `apmField` should be provided. The `apmField` is set to 'host.hostname'.

```tsx
<LinkToApmServices hostName={'host name here'} apmField={'host.hostname'} />
```

The link should look similar to

`/base-path/apm/services?comparisonEnabled=true&environment=ENVIRONMENT_ALL&kuery=host.hostname:"host name here"&rangeFrom=now-15m&rangeTo=now&offset=1d`
