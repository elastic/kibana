# ES|QL Read Query Templates — Records, Timeline, Influencers, RCA

Use `platform.core.execute_esql` with the query text below. Replace `?param` placeholders with actual values.

---

## ad_query_anomaly_records

Search anomaly records by score threshold and time range. Use `job_id_pattern='*'` for cross-job search or an exact job ID for single-job drill-down.

**Parameters:** `job_id_pattern` (string, LIKE wildcards), `min_score` (number), `start_time` (ISO 8601), `end_time` (ISO 8601)

```esql
FROM .ml-anomalies-*
| WHERE result_type == "record"
  AND job_id LIKE ?job_id_pattern
  AND record_score >= ?min_score
  AND timestamp >= ?start_time
  AND timestamp <= ?end_time
| SORT record_score DESC
| LIMIT 50
| KEEP job_id, timestamp, record_score, function, field_name, actual, typical,
       by_field_name, by_field_value, over_field_name, over_field_value,
       partition_field_name, partition_field_value, multi_bucket_impact,
       initial_record_score, detector_index, probability
```

## ad_query_anomaly_timeline

Time-series summary of anomaly severity across jobs. Use for cross-job swimlanes and detecting anomaly storms.

**Parameters:** `job_id_pattern`, `min_score`, `start_time`, `end_time`

```esql
FROM .ml-anomalies-*
| WHERE result_type == "bucket"
  AND job_id LIKE ?job_id_pattern
  AND anomaly_score >= ?min_score
  AND timestamp >= ?start_time
  AND timestamp <= ?end_time
| STATS max_score = MAX(anomaly_score),
        job_count = COUNT_DISTINCT(job_id),
        jobs = VALUES(job_id),
        composite_score = SUM(anomaly_score),
        avg_score = AVG(anomaly_score)
  BY timestamp
| SORT timestamp
```

## ad_query_influencers

Find the most unusual entities across all or selected jobs. When `job_count > 1`, entities appear as influencers in multiple jobs — prime RCA signal.

**Parameters:** `job_id_pattern`, `min_score`, `start_time`, `end_time`

```esql
FROM .ml-anomalies-*
| WHERE result_type == "influencer"
  AND job_id LIKE ?job_id_pattern
  AND influencer_score >= ?min_score
  AND timestamp >= ?start_time
  AND timestamp <= ?end_time
| STATS total_score = SUM(influencer_score),
        job_count = COUNT_DISTINCT(job_id),
        jobs = VALUES(job_id),
        max_score = MAX(influencer_score)
  BY influencer_field_name, influencer_field_value
| SORT total_score DESC
| LIMIT 30
```

## ad_rca_blast_radius

Measure how widespread an issue is by counting distinct partitions/entities affected by a specific anomalous value.

**Parameters:** `anomalous_value` (string), `min_score`, `start_time`, `end_time`

```esql
FROM .ml-anomalies-*
| WHERE result_type == "record"
  AND record_score >= ?min_score
  AND timestamp >= ?start_time
  AND timestamp <= ?end_time
  AND (by_field_value == ?anomalous_value
       OR over_field_value == ?anomalous_value
       OR partition_field_value == ?anomalous_value)
| STATS affected_partitions = COUNT_DISTINCT(partition_field_value),
        affected_jobs = COUNT_DISTINCT(job_id),
        total_anomalies = COUNT(*),
        max_score = MAX(record_score),
        time_span_hours = DATE_DIFF("hour", MIN(timestamp), MAX(timestamp))
  BY by_field_value
| SORT affected_partitions DESC
```

## ad_rca_correlation

Find temporally correlated anomalies across different jobs. Sort by timestamp to detect cascading failures (network → app → DB).

**Parameters:** `job_id_pattern`, `min_score`, `start_time`, `end_time`

```esql
FROM .ml-anomalies-*
| WHERE result_type == "record"
  AND job_id LIKE ?job_id_pattern
  AND record_score >= ?min_score
  AND timestamp >= ?start_time
  AND timestamp <= ?end_time
| SORT timestamp ASC
| KEEP job_id, timestamp, record_score, function, field_name,
       by_field_name, by_field_value, partition_field_name, partition_field_value,
       over_field_name, over_field_value, multi_bucket_impact
| LIMIT 100
```

## ad_rca_cross_job_entity_match

Starting from a specific entity value, find all other jobs where that entity appears as anomalous — across all split field types.

**Parameters:** `entity_value` (string), `min_score`, `start_time`, `end_time`

```esql
FROM .ml-anomalies-*
| WHERE result_type == "record"
  AND record_score >= ?min_score
  AND timestamp >= ?start_time
  AND timestamp <= ?end_time
  AND (partition_field_value == ?entity_value
       OR by_field_value == ?entity_value
       OR over_field_value == ?entity_value)
| STATS max_score = MAX(record_score),
        anomaly_count = COUNT(*),
        functions = VALUES(function),
        fields = VALUES(field_name),
        first_anomaly = MIN(timestamp),
        last_anomaly = MAX(timestamp)
  BY job_id
| SORT max_score DESC
```

## ad_rca_multi_job_entities

Find entities that are anomalous in multiple jobs simultaneously — the strongest root cause signal.

**Parameters:** `min_score`, `min_job_count` (number, use 2 for root cause candidates), `start_time`, `end_time`

```esql
FROM .ml-anomalies-*
| WHERE result_type == "record"
  AND record_score >= ?min_score
  AND timestamp >= ?start_time
  AND timestamp <= ?end_time
| STATS job_count = COUNT_DISTINCT(job_id),
        jobs = VALUES(job_id),
        max_score = MAX(record_score),
        total_records = COUNT(*),
        functions = VALUES(function),
        fields = VALUES(field_name)
  BY partition_field_value
| WHERE job_count >= ?min_job_count
| SORT job_count DESC, max_score DESC
| LIMIT 20
```

## ad_rca_source_evidence

After identifying an anomaly, retrieve raw source documents from the original data index.

**Parameters:** `source_index` (LIKE pattern, get from datafeed config), `start_time`, `end_time`

```esql
FROM * METADATA _index
| WHERE _index LIKE ?source_index
  AND @timestamp >= ?start_time
  AND @timestamp <= ?end_time
| SORT @timestamp DESC
| LIMIT 50
```
