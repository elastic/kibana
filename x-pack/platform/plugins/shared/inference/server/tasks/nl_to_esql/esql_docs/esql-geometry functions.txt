# ST_X

Extracts the `x` coordinate from the supplied point. For points of type `geo_point`, this is equivalent to extracting the longitude value.

## Syntax

`ST_X(point)`

### Parameters

#### point

Expression of type `geo_point` or `cartesian_point`. If `null`, the function returns `null`.

## Examples

The following query creates a geo point from a WKT string, then extracts both the x (longitude) and y (latitude) coordinates from it:

```esql
ROW point = TO_GEOPOINT("POINT(42.97109629958868 14.7552534006536)")
| EVAL x = ST_X(point), y = ST_Y(point)
```

# ST_Y

Extracts the `y` coordinate from the supplied point. For points of type `geo_point`, this is equivalent to extracting the latitude value.

## Syntax

`ST_Y(point)`

### Parameters

#### point

Expression of type `geo_point` or `cartesian_point`. If `null`, the function returns `null`.

## Examples

This query demonstrates how to extract the x (longitude) and y (latitude) coordinates from a geo point created from a WKT string:

```esql
ROW point = TO_GEOPOINT("POINT(42.97109629958868 14.7552534006536)")
| EVAL x = ST_X(point), y = ST_Y(point)
```

# ST_NPOINTS

Counts the number of points in the supplied geometry.

## Syntax

`ST_NPOINTS(geometry)`

### Parameters

#### geometry

Expression of type `geo_point`, `geo_shape`, `cartesian_point`, or `cartesian_shape`. If `null`, the function returns `null`.

## Examples

This query retrieves the city boundary for the airport with abbreviation "CPH", counts the number of points that make up the city boundary geometry, and returns the abbreviation, airport name, and point count:

```esql
FROM airport_city_boundaries
| WHERE abbrev == "CPH"
| EVAL points = ST_NPOINTS(city_boundary)
| KEEP abbrev, airport, points
```

# ST_SIMPLIFY

Simplifies the input geometry by applying the Douglas-Peucker algorithm with a specified tolerance. Vertices within the tolerance distance from the simplified shape are removed. The resulting geometry may be invalid, even if the original input was valid.

## Syntax

`ST_SIMPLIFY(geometry, tolerance)`

### Parameters

#### geometry

Expression of type `geo_point`, `geo_shape`, `cartesian_point`, or `cartesian_shape`. If `null`, the function returns `null`.

#### tolerance

Optional. Tolerance for the geometry simplification, in the units of the input SRS.

## Examples

This query creates a polygon geometry from a WKT string and then simplifies it using a tolerance value of 0.7, reducing the number of vertices in the polygon:

```esql
ROW wkt = "POLYGON ((7.998 53.827, 9.470 53.068, 15.754 53.801, 16.523 57.160, 11.162 57.868, 8.064 57.445, 6.219 55.317, 7.998 53.827))"
| EVAL simplified = ST_SIMPLIFY(TO_GEOSHAPE(wkt), 0.7)
```
