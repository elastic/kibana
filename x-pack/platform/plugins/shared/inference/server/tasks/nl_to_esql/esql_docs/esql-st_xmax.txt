# ST_XMAX

Extracts the maximum value of the `x` coordinates from the supplied geometry. For geometries of type `geo_point` or `geo_shape`, this is equivalent to extracting the maximum `longitude` value.

## Syntax

`ST_XMAX(point)`

### Parameters

#### point

Expression of type `geo_point`, `geo_shape`, `cartesian_point`, or `cartesian_shape`. If the value is `null`, the function returns `null`.

## Examples

Extract the minimum and maximum x and y coordinates from the city boundary of the airport with abbreviation "CPH":

```esql
FROM airport_city_boundaries
| WHERE abbrev == "CPH"
| EVAL envelope = ST_ENVELOPE(city_boundary)
| EVAL xmin = ST_XMIN(envelope), xmax = ST_XMAX(envelope), ymin = ST_YMIN(envelope), ymax = ST_YMAX(envelope)
| KEEP abbrev, airport, xmin, xmax, ymin, ymax
```

## Limitations

- Spatial types are not supported in the `SORT` processing command. Specifying a column of type `geo_point`, `geo_shape`, `cartesian_point`, or `cartesian_shape` as a sort parameter will result in an error.
