# TO_IP

Converts an input string to an IP value. If the conversion fails, the result is a `null` value and a warning header is added to the response with details about the failure.

## Syntax

`TO_IP(field, options, leading_zeros)`

### Parameters

#### field

Input value. The input can be a single- or multi-valued column or an expression.

#### options

(Optional) Additional options.

#### leading_zeros

(keyword) Specifies how to handle leading zeros in IPv4 addresses.

## Examples

Convert two string values to IP addresses, where one is valid and the other is invalid, and filter for IPs matching the CIDR range "1.0.0.0/8".
```esql
ROW str1 = "1.1.1.1", str2 = "foo"
| EVAL ip1 = TO_IP(str1), ip2 = TO_IP(str2)
| WHERE CIDR_MATCH(ip1, "1.0.0.0/8")
```
Convert valid and invalid IP strings. The invalid conversion returns `null` and triggers a warning header.

Convert a string with leading zeros in the IPv4 address to an IP, interpreting the zeros as octal numbers.
```esql
ROW s = "1.1.010.1" | EVAL ip = TO_IP(s, {"leading_zeros":"octal"})
```
Parse an IPv4 address with leading zeros as octal, similar to `ping` or `ftp`.

Convert a string with leading zeros in the IPv4 address to an IP, interpreting the zeros as decimal numbers.
```esql
ROW s = "1.1.010.1" | EVAL ip = TO_IP(s, {"leading_zeros":"decimal"})
```
Parse an IPv4 address with leading zeros as decimal, following Java's `InetAddress.getByName`.
