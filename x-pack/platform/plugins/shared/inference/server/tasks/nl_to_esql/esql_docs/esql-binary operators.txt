# BINARY OPERATORS

Binary operators are used to compare values or perform arithmetic operations between two fields or values. If either field is multivalued, the result is `null`.

## Syntax

`<field or value> <operator> <field or value>`

### Parameters

#### <field or value>

The left-hand side operand for the binary operation.

#### <operator>

The binary operator to use. Supported operators include:
- Equality `==`
- Inequality `!=`
- Less than `<`
- Less than or equal to `<=`
- Greater than `>`
- Greater than or equal to `>=`
- Add `+`
- Subtract `-`
- Multiply `*`
- Divide `/`
- Modulus `%`

#### <field or value>

The right-hand side operand for the binary operation.

## Examples

Filters employees whose department is "Engineering".

```esql
FROM employees
| WHERE department == "Engineering"
```

Filters employees whose salary is not 50,000.

```esql
FROM employees
| WHERE salary != 50000
```

Filters employees younger than 30.

```esql
FROM employees
| WHERE age < 30
```

Filters employees with five or fewer years of experience.

```esql
FROM employees
| WHERE years_experience <= 5
```

Filters employees with a salary greater than 100,000.

```esql
FROM employees
| WHERE salary > 100000
```

Filters employees aged 40 or older.

```esql
FROM employees
| WHERE age >= 40
```

Creates a new column `total` by adding the `salary` and `bonus` fields.

```esql
FROM employees
| EVAL total = salary + bonus
```

Creates a new column `difference` by subtracting `deductions` from `salary`.

```esql
FROM employees
| EVAL difference = salary - deductions
```

Creates a new column `double_salary` by multiplying `salary` by 2.

```esql
FROM employees
| EVAL double_salary = salary * 2
```

Creates a new column `ratio` by dividing `salary` by 1,000.

```esql
FROM employees
| EVAL ratio = salary / 1000
```

Creates a new column `remainder` by calculating the remainder of `salary` divided by 1,000.

```esql
FROM employees
| EVAL remainder = salary % 1000
```

## Limitations

- If either field is multivalued, the result is `null`.
- For division `/`, dividing two integer types yields an integer result, rounding towards 0. For floating point division, cast one of the arguments to `DOUBLE`.
- Comparisons are pushed to the underlying search index only if one side is a constant and the other is a field in the index with both a mapping-index and doc-values.
