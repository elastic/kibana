# ABSENT

The ABSENT function returns true if the input expression yields no non-null values within the current aggregation context; otherwise, it returns false.

## Syntax

`ABSENT(field)`

### Parameters

#### field

Expression that outputs values to be checked for absence.

## Examples

This query checks whether the `languages` field is absent (i.e., contains only null values) for the employee with `emp_no` 10020. The result is returned as a boolean value in the `is_absent` column.

```esql
FROM employees
| WHERE emp_no == 10020
| STATS is_absent = ABSENT(languages)
```

This query determines, for each group of employees sharing the same `languages` value, whether the `salary` field is absent (contains only null values) within that group. The result is shown in the `is_absent` column for each language group.

```esql
FROM employees
| STATS is_absent = ABSENT(salary) BY languages
```

This query checks if the `languages` field is absent for the employee with `emp_no` 10020. It then converts the boolean result to an integer, returning 1 if absent and 0 otherwise, in the `is_absent` column.

```esql
FROM employees
| WHERE emp_no == 10020
| STATS is_absent = TO_INTEGER(ABSENT(languages))
```
