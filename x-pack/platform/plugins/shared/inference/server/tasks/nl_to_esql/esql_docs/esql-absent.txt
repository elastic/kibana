# ABSENT

The ABSENT function returns true if the input expression yields no non-null values within the current aggregation context; otherwise, it returns false.

## Syntax

`ABSENT(field)`

### Parameters

#### field

Expression that outputs values to be checked for absence.

## Examples

The following query checks if the `languages` field is absent (i.e., contains only null values) for the employee with employee number 10020. The result is a boolean value indicating the absence.

```esql
FROM employees
| WHERE emp_no == 10020
| STATS is_absent = ABSENT(languages)
```

This query checks for the absence of the `salary` field within each group of employees sharing the same `languages` value. For each group, it returns whether the `salary` field is absent.

```esql
FROM employees
| STATS is_absent = ABSENT(salary) BY languages
```

This query returns 1 if the `languages` field is absent for employee number 10020, and 0 otherwise. It uses the `TO_INTEGER` function to convert the boolean result of `ABSENT(languages)` into an integer.

```esql
FROM employees
| WHERE emp_no == 10020
| STATS is_absent = TO_INTEGER(ABSENT(languages))
```
