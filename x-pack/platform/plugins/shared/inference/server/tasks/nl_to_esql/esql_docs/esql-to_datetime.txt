# TO_DATETIME

Converts an input value to a date value. A string will only be successfully converted if it matches the format `yyyy-MM-dd'T'HH:mm:ss.SSS'Z'`. For other date formats, use `DATE_PARSE`. When converting from nanosecond resolution to millisecond resolution, the nanosecond date is truncated, not rounded.

## Syntax

`TO_DATETIME(field)`

### Parameters

#### field

Input value to convert. The input can be a single- or multi-valued column or an expression.

## Examples

Convert a multi-valued string column to datetime, where only values matching the required format are converted:

```esql
ROW string = ["1953-09-02T00:00:00.000Z", "1964-06-02T00:00:00.000Z", "1964-06-02 00:00:00"]
| EVAL datetime = TO_DATETIME(string)
```

This query creates a row with a column named `string` containing three date strings. It then creates a new column `datetime` by converting each string to a datetime value, but only strings matching the exact format `yyyy-MM-dd'T'HH:mm:ss.SSS'Z'` are converted; others result in `null`.

If a value does not match the required date format, the conversion results in a `null` value and a warning header is added to the response.

Convert numeric values interpreted as milliseconds since the Unix epoch to datetime:

```esql
ROW int = [0, 1]
| EVAL dt = TO_DATETIME(int)
```

This query creates a row with a column named `int` containing the values 0 and 1, which represent milliseconds since the Unix epoch. It then creates a new column `dt` by converting these numeric values into datetime values.

## Limitations

- Only strings matching the format `yyyy-MM-dd'T'HH:mm:ss.SSS'Z'` are converted; others result in `null`.
- When converting from nanosecond to millisecond resolution, the value is truncated, not rounded.
