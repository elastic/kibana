## Task
For each criterion listed below, decide whether the provided **output** satisfies the requirement, fails to satisfy it, or is not applicable for the given example.

Return your decision by calling the **score** tool exactly once.

Each decision must be one of:
- **PASS** – the output clearly meets the criterion.
- **FAIL** – the output clearly violates or misses the criterion.
- **N/A** – the criterion does not apply to this example.

## Scope of evaluation

Evaluate the criteria as written. Decide each criterion against the literal scope it asserts — do **not** infer additional requirements that the criterion does not state.

In particular, when a criterion asserts that a *call* was made (for example: "tool is called with X", "queries Y", "invokes Z"), the criterion is about the call itself. **PASS** when the call was made as described, regardless of whether the call returned data, returned an empty result, or surfaced a "not found" error. Do not require successful resolution, non-empty results, or downstream behavior.

**Exception:** if the criterion explicitly requires success or a specific return value (e.g. "successfully executes X", "returns Y", "catches Z"), the return matters — evaluate resolution as well.

When a criterion is **disjunctive** (uses "or" / "OR" between alternatives, e.g. "do X **or** clearly state Y is unavailable"), satisfying **either** branch is sufficient. **PASS** when any one branch is met. Do not require both branches, and do not treat the unfulfilled branch as a "partial failure". Treat conjunctive criteria ("X **and** Y") the opposite way: both parts must hold to PASS.
