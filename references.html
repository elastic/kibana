HippoRAG: Neurobiologically Inspired Long-Term Memory for Large Language Models

2025 Bernal Jiménez Gutiérrez (The Ohio State University), Yiheng Shu (The Ohio State University), Yu Gu (The Ohio State University), Michihiro Yasunaga (Stanford University), Yu Su (The Ohio State University)
2405.14831v3.pdf Local PDF ↗ arXiv ↗
▸

HippoRAG is a retrieval-augmented generation framework inspired by the hippocampal indexing theory of human long-term memory that enables LLMs to integrate knowledge across passage boundaries. It synergistically combines LLMs, knowledge graphs, and the Personalized PageRank algorithm to mimic the roles of the neocortex, parahippocampal regions, and hippocampus. HippoRAG outperforms state-of-the-art RAG methods by up to 20% on multi-hop QA benchmarks while being 10-30 times cheaper and 6-13 times faster than iterative retrieval methods.

Problem being solved: Current retrieval-augmented generation (RAG) methods fail at tasks requiring knowledge integration across multiple passages because each passage is encoded in isolation. Multi-hop question answering, scientific literature review, legal case briefing, and medical diagnosis all require connecting information spread across disparate documents. Iterative multi-step retrieval approaches can partially address this but are expensive, slow, and still fail on harder "path-finding" queries where many possible reasoning paths exist. A deeper challenge is that no existing RAG system can efficiently associate disparate facts the way human memory does, leaving a significant capability gap for real-world knowledge integration tasks.

Approach: HippoRAG is inspired by the hippocampal memory indexing theory, which explains how the human brain achieves pattern separation and pattern completion through interactions between the neocortex, parahippocampal regions (PHR), and hippocampus. The key insight is to map these biological components onto computational counterparts: an LLM serves as the neocortex to extract knowledge, dense retrieval encoders serve as PHR to detect synonymy, and a schemaless knowledge graph indexed with Personalized PageRank (PPR) serves as the hippocampal index. PPR biased toward query-derived seed nodes allows multi-hop associations to be traversed in a single retrieval step, rather than requiring iterative LLM calls.

Methodology: During offline indexing, an instruction-tuned LLM extracts open knowledge graph (KG) triples from each passage via OpenIE, building a schemaless hippocampal index. Dense retrieval encoders add synonymy edges between semantically similar noun phrases above a cosine similarity threshold. During online retrieval, the LLM extracts named entities from the query, which are linked to KG nodes via the encoder. Personalized PageRank is then run on the KG with those nodes as seeds, and the resulting node probability distribution is aggregated over passages for ranking. A node specificity score (inverse of how many passages a node appears in) further refines the seed weights as a neurobiologically plausible IDF signal.

Results: On MuSiQue and 2WikiMultiHopQA benchmarks, HippoRAG improves single-step retrieval R@2 by approximately 3% and 20% respectively over the strongest baselines, and the all-recall metric (all supporting passages retrieved) improves even more dramatically, by up to 38 points on 2WikiMultiHopQA. Single-step HippoRAG matches or exceeds iterative IRCoT retrieval while being 10-30 times cheaper and 6-13 times faster. Combining HippoRAG with IRCoT yields further complementary gains of up to 4% and 18% on MuSiQue and 2WikiMultiHopQA. QA F1 scores improve by up to 17 points on 2WikiMultiHopQA, and HippoRAG is the only tested method capable of handling path-finding multi-hop questions that require associating information across many possible entity paths.

TL;DR: HippoRAG models human hippocampal memory using LLM-extracted knowledge graphs and Personalized PageRank to enable single-step multi-hop retrieval, dramatically outperforming existing RAG methods on knowledge integration tasks while being far cheaper and faster than iterative alternatives.

M+: Extending MemoryLLM with Scalable Long-Term Memory

2025 Yu Wang (UC San Diego), Dmitry Krotov (MIT-IBM Watson Lab, IBM Research), Yuanzhe Hu (UC San Diego), Yifan Gao (Amazon), Wangchunshu Zhou (OPPO), Julian McAuley (UC San Diego), Dan Gutfreund (MIT-IBM Watson Lab, IBM Research), Rogerio Feris (MIT-IBM Watson Lab, IBM Research), Zexue He (UC San Diego, IBM Research)
2502.00592v2.pdf Local PDF ↗ arXiv ↗
▸

M+ addresses the limited long-term retention of latent-space memory models by extending MemoryLLM with a scalable long-term memory mechanism paired with a co-trained retriever. The approach stores dropped memory tokens on CPU rather than discarding them, then retrieves relevant ones at generation time using a learned projector-based retriever. M+ extends effective knowledge retention from under 20k to over 160k tokens while maintaining a lower or comparable GPU memory footprint versus strong baselines.

Problem being solved: Large language models augmented with latent-space memory can extend their effective context window without excessive GPU overhead, but existing methods struggle to retain information from the distant past. MemoryLLM, a representative latent-space memory model, compresses past information into hidden states forming a 1B-parameter memory pool, yet can only reliably retain knowledge for up to about 20k tokens. Tokens that are dropped from the short-term memory pool during updates are permanently discarded, creating an irrecoverable loss of earlier context. Token-level (text-based) memory alternatives are less compressed and harder to train end-to-end, so a better latent-space solution is needed for truly long-horizon tasks.

Approach: M+ augments MemoryLLM with a long-term memory (LTM) store that captures tokens dropped from the short-term memory pool instead of discarding them. A lightweight co-trained retriever, consisting of query and key projectors (two-layer MLPs projecting to d/20 dimensions), dynamically fetches the most relevant LTM tokens during generation using dot-product similarity in hidden-state space. Because retrieval happens once per layer for all query heads rather than per-head per-layer as in KV-cache methods, the added latency is modest. The LTM is kept on CPU and only loaded to GPU as needed, so GPU memory overhead is not increased relative to the baseline MemoryLLM.

Methodology: M+ is built on Llama-3.1-8B and trained in three curriculum stages: Stage 1 continually trains the short-term memory model on fineweb-edu (1.2M steps) to handle short documents; Stage 2 adds long-document training using SlimPajama excerpts of 4k-64k tokens to improve long-context modeling; Stage 3 introduces the long-term memory and co-trains the retriever on further long-document examples. A multi-LoRA design uses separate LoRA weights for the update (write) and generation (read) passes. The model uses 10,240 short-term memory tokens per layer and retrieves 2,560 tokens from LTM per layer at inference, keeping total effective memory constant at 12,800 tokens. The LTM is capped at 150k tokens and evicts the oldest entries when full.

Results: On LongBook-QA (avg. 192k-token books) and the newly introduced LongBook-Event-QA benchmark, M+ outperforms all baselines including Llama-3.1-8B with BM25 retrieval, SnapKV, and the smaller Llama-3.2-3B-128k model, despite processing fewer tokens. On knowledge retention benchmarks (SQuAD, NaturalQA), M+ retains accurate recall beyond 160k injected tokens, compared to roughly 20k for MemoryLLM-7B and roughly 30k for Llama-3.1-8B-SnapKV given a 48k context. GPU memory usage for M+ (with CPU offloading) is lower than all non-offloaded baselines at 17,973 MB. On shorter-context LongBench tasks, M+ matches Llama-3.1-8B on 4 of 6 datasets, with minor degradation on multi-hop tasks attributable to the random dropping mechanism.

TL;DR: M+ extends MemoryLLM by saving dropped latent memory tokens to CPU-based long-term storage and retrieving them with a co-trained projector-based retriever, pushing reliable knowledge retention from 20k to over 160k tokens at no additional GPU memory cost.

Mem0: Building Production-Ready AI Agents with Scalable Long-Term Memory

2025 Prateek Chhikara (mem0.ai), Dev Khant (mem0.ai), Saket Aryan (mem0.ai), Taranjeet Singh (mem0.ai), Deshraj Yadav (mem0.ai)
2504.19413v1.pdf Local PDF ↗ arXiv ↗
▸

LLMs are fundamentally limited by fixed context windows that prevent coherent long-term interactions across sessions. This paper introduces Mem0, a scalable memory architecture that dynamically extracts, consolidates, and retrieves salient information from conversations, with an enhanced graph-based variant for relational reasoning. Evaluated on the LOCOMO benchmark, Mem0 outperforms all baselines while achieving 91% lower p95 latency and over 90% token cost reduction compared to full-context approaches.

Problem being solved: Large language models reset their knowledge at each session boundary because they rely on fixed context windows, making them unable to maintain coherent, personalized interactions over extended multi-session dialogues. Even models with large context windows (e.g., 200K tokens) face two compounding issues: conversation history eventually exceeds any finite limit, and thematically discontinuous exchanges bury critical facts among irrelevant tokens. Attention mechanisms also degrade over distant tokens, further eroding recall quality. This forces AI agents to forget user preferences, repeat questions, and contradict previously established facts, fundamentally undermining user trust in high-stakes domains like healthcare and enterprise support.

Approach: The paper introduces Mem0, a memory-centric architecture that continuously extracts salient facts from ongoing conversations, consolidates them into a compact knowledge base, and retrieves only relevant memories at query time. An enhanced variant, Mem0g, augments the natural language memory store with a directed labeled knowledge graph where entities are nodes and relationships are labeled edges, enabling richer relational and temporal reasoning. Rather than storing raw conversation chunks, both systems distill conversations into concise, structured representations that dramatically reduce token consumption while preserving the information needed to answer complex questions.

Methodology: Mem0 processes incoming message pairs through two sequential phases: an extraction phase that uses an LLM to identify salient facts from the new exchange given a rolling conversation summary and recent message window, and an update phase that compares each candidate fact to semantically similar existing memories and issues one of four operations (ADD, UPDATE, DELETE, NOOP) via LLM function-calling. Mem0g adds a two-stage graph pipeline that extracts entity-type pairs and relationship triplets using GPT-4o-mini, stores them in Neo4j, and employs a dual retrieval strategy combining entity-centric subgraph traversal with semantic triplet matching. Both systems were evaluated on the LOCOMO benchmark, which contains 10 extended conversations averaging 600 dialogues and 26,000 tokens each, with around 200 questions per conversation spanning single-hop, multi-hop, temporal, and open-domain categories. Six baseline categories were tested, including RAG with varying chunk sizes, full-context processing, OpenAI's memory feature, Zep, LangMem, and prior LOCOMO benchmarks.

Results: Mem0 achieves state-of-the-art LLM-as-a-Judge scores across most question types, delivering 5%, 11%, and 7% relative improvements in single-hop, temporal, and multi-hop categories over the best-performing prior methods. Mem0g further improves temporal and open-domain performance, surpassing the base variant by roughly 2% overall. Compared to the full-context baseline, Mem0 reduces p95 latency by 91% (from 17 seconds to 1.44 seconds) and cuts token consumption by over 90%, using only 7k tokens per conversation versus 26k for full-context. Mem0 also delivers a 26% relative improvement in LLM-as-a-Judge score over OpenAI's memory feature and outperforms all RAG configurations by roughly 10% relative on the overall quality metric.

TL;DR: Mem0 introduces a scalable, production-ready long-term memory system for LLM agents that extracts and consolidates conversational facts into compact representations, achieving state-of-the-art accuracy on the LOCOMO benchmark while cutting latency and token costs by over 90% compared to full-context approaches.

Seeing, Listening, Remembering, and Reasoning: A Multimodal Agent with Long-Term Memory

2025 Lin Long (ByteDance Seed, Zhejiang University), Yichen He (ByteDance Seed), Wentao Ye (ByteDance Seed, Zhejiang University), Yiyuan Pan (ByteDance Seed, Shanghai Jiao Tong University), Yuan Lin (ByteDance Seed), Hang Li (ByteDance Seed), Junbo Zhao (Zhejiang University), Wei Li (ByteDance Seed)

▸

This paper presents M3-Agent, a multimodal agent framework with long-term memory that continuously processes real-time video and audio streams to build episodic and semantic memories organized in an entity-centric graph. The approach enables agents to accumulate world knowledge and perform multi-turn reasoning over memory to complete complex tasks. Trained via reinforcement learning, M3-Agent outperforms the strongest prompting-based baseline by 6.7%, 7.7%, and 5.3% on three benchmarks including a newly introduced evaluation suite.

Problem being solved: Existing multimodal agents lack the ability to form and reason over long-term memory in the way humans do, limiting their usefulness in real-world settings such as household robots. Current approaches either extend context windows for finite offline videos or compress visual tokens, neither of which scales to arbitrarily long streams. Language-based memory descriptions also suffer from inconsistency and ambiguity when tracking entities such as characters across long time spans. There is also no suitable benchmark to evaluate higher-level cognitive capabilities such as person understanding, general knowledge extraction, and cross-modal reasoning over long-term memory.

Approach: The authors propose M3-Agent, a framework with two parallel processes: memorization and control. During memorization, the agent processes a continuous video stream clip-by-clip, generating episodic memory (concrete observed events) and semantic memory (general world knowledge such as character identities and attributes). Entity consistency is maintained through facial recognition and speaker diarization tools that assign persistent identifiers to characters, anchoring all textual descriptions to stable multimodal nodes in a graph. During control, the agent uses reinforcement learning to perform multi-turn, iterative memory retrieval rather than single-turn RAG, enabling more flexible and accurate task completion.

Methodology: Long-term memory is stored as an entity-centric multimodal graph where nodes represent text, image, or audio memory items linked by edges that encode relationships. The memorization model is initialized from Qwen2.5-Omni-7b and trained via imitation learning on synthetic memory demonstrations generated by a combination of Gemini-1.5-Pro and GPT-4o. The control model is initialized from Qwen3-32b and trained using the DAPO reinforcement learning algorithm, receiving binary rewards from a GPT-4o evaluator. To evaluate the framework, the authors introduce M3-Bench, consisting of 100 robot-perspective videos and 920 web-sourced YouTube videos with 4,490 manually annotated open-ended QA pairs across five question types.

Results: M3-Agent achieves accuracy of 30.7%, 48.9%, and 61.8% on M3-Bench-robot, M3-Bench-web, and VideoMME-long respectively, surpassing the strongest baseline (Gemini-GPT4o-Hybrid) by 6.7%, 7.7%, and 5.3%. Ablation studies show that removing semantic memory degrades accuracy by 17.1%, 19.2%, and 13.1% across the three benchmarks, and removing RL training reduces accuracy by 10.0%, 8.0%, and 9.3%. Inter-turn instruction and reasoning mode are also both shown to be critical components. The SFT memorization model (memory-7b-sft) outperforms even the prompted Gemini-1.5-Pro baseline for memory generation on most benchmarks.

TL;DR: M3-Agent is a multimodal agent that builds entity-centric, episodic and semantic long-term memory from continuous video and audio streams, then uses reinforcement-learning-trained multi-turn reasoning to answer questions, consistently outperforming strong prompted commercial model baselines across three long-video benchmarks.

A Multi-Memory Segment System for Generating High-Quality Long-Term Memory Content in Agents

2026 Gaoke Zhang (Tianjin University), Bo Wang (Tianjin University), Yunlong Ma (Tianjin University), Dongming Zhao (China Mobile Communication Group Tianjin Co., Ltd), Zifei Yu (Huizhi Xingyuan Information Technology Co., Ltd)
2508.15294v3.pdf Local PDF ↗ arXiv ↗
▸

Existing LLM agent memory systems store only simple summaries of dialogues, producing low-quality memory content that degrades retrieval and response performance. This paper proposes a Multi-Memory Segment system (MMS) grounded in cognitive psychology that converts short-term memory into multiple typed long-term memory segments, then builds separate retrieval and contextual memory units from them. Experiments on the LoCoMo dataset show MMS substantially outperforms prior methods in both recall and answer quality while being faster and more token-efficient than A-MEM.

Problem being solved: Agent memory systems for LLMs have focused heavily on retrieval mechanisms while largely neglecting the quality of the stored memory content itself. Prevailing approaches such as A-MEM and MemoryBank reduce dialogue history to simple summaries and keyword lists, which creates a semantic gap between the user's query and the stored memory. This gap limits retrieval recall and degrades the quality of generated responses. Human long-term memory formation is inherently multi-dimensional, involving episodic, semantic, and conceptual components, but no prior agent memory system exploits this richness in a principled way.

Approach: The authors draw on three cognitive psychology theories: Tulving's multiple memory systems theory (episodic and semantic memory), the levels-of-processing theory (deeper encoding leads to stronger memory), and the encoding specificity principle (retrieval is most effective when encoding context matches query context). These theories motivate extracting four distinct memory segments from each short-term memory chunk: keywords, multiple cognitive perspectives, episodic memory (event-based), and semantic memory (factual knowledge points). Two separate memory unit types are then constructed: retrieval memory units optimized for query matching, and contextual memory units optimized for knowledge-enriched response generation.

Methodology: Given a round of dialogue as short-term memory, an LLM prompt extracts keywords, multiple cognitive perspectives, episodic memory (situations), and semantic memory (knowledge points) as structured JSON. Retrieval memory units combine keywords, the original short-term memory, cognitive perspectives, and episodic memory, and are embedded using all-MiniLM-L6-v2 for cosine-similarity-based retrieval. Contextual memory units substitute semantic memory for episodic memory, as semantic knowledge points better enrich generation without introducing redundancy. The top-k retrieved contextual units are provided as context at inference time. Experiments used GPT-4o, Qwen2.5-14B, and Gemini-2.5-pro-preview on the LoCoMo dataset, evaluating Recall@1/3/5, F1, and BLEU-1 across five question types, along with ablation, robustness, and overhead analyses.

Results: MMS outperforms NaiveRAG, MemoryBank, and A-MEM in the majority of tasks and metrics across all three base models. The most pronounced gains appear in multi-hop reasoning, where GPT-4o and Gemini improve recall by 8 to 11 points over A-MEM, and in open-domain and temporal tasks. Ablation studies confirm that each memory segment contributes positively, with combined removal of cognitive perspectives and episodic memory causing the largest drops. Despite producing substantially richer memory content, MMS achieves lower average latency (1.31 s) and fewer tokens (744) per query than A-MEM (3.93 s, 1429 tokens), demonstrating practical efficiency alongside quality gains.

TL;DR: MMS improves LLM agent long-term memory by replacing simple dialogue summaries with cognitively grounded multi-segment memory units, yielding significantly better retrieval recall and answer quality at lower computational cost than prior methods.

SGMem: Sentence Graph Memory for Long Term Conversational Agents

2025 Yaxiong Wu (Huawei Technologies Co., Ltd), Yongyue Zhang (Huawei Technologies Co., Ltd), Sheng Liang (Huawei Technologies Co., Ltd), Yong Liu (Huawei Technologies Co., Ltd)
2509.21212v1.pdf Local PDF ↗ arXiv ↗
▸

Long-term conversational agents struggle to retrieve relevant information across multi-session dialogue histories due to memory fragmentation. SGMem addresses this by organizing dialogue as sentence-level graphs that link raw dialogue to generated memory, enabling coherent multi-hop retrieval. The approach consistently outperforms strong baselines on the LongMemEval and LoCoMo benchmarks.

Problem being solved: Long-term conversational agents must manage dialogue histories that exceed LLM context windows, requiring external memory systems. Existing methods based on fact extraction, summarization, and reflection reduce redundancy but fragment information across raw dialogue and derived snippets at coarse granularities such as turns, rounds, or sessions. This memory fragmentation makes it difficult to retrieve coherent and relevant context for answering questions, especially those requiring multi-session or temporal reasoning. Graph-based RAG methods relying on entity-relation extraction partially address structure but are computationally expensive and still lose contextual richness. There is a need for a lightweight, fine-grained memory organization that bridges raw dialogue and generated memory.

Approach: SGMem (Sentence Graph Memory) structures dialogue at the sentence level, treating each sentence as a node in a graph within chunked dialogue units such as turns, rounds, or sessions. Sentences within and across chunks are connected via membership edges and k-nearest-neighbor similarity edges computed from dense embeddings, enabling explicit modeling of semantic associations without LLM-based entity extraction. During retrieval, candidate sentences are expanded via n-hop graph traversal, mapped back to their parent chunks, and ranked by aggregate similarity scores. Retrieved chunks are then combined with generated memory including summaries, facts, and insights to form a coherent context for response generation. This design requires only standard sentence segmentation tools such as NLTK, making it lightweight and practical.

Methodology: SGMem construction involves decomposing sessions into rounds, turns, and sentences using NLTK, then having an LLM generate summaries, facts, and insights. All memory units are embedded with Sentence-BERT (all-MiniLM-L6-v2) and indexed in seven vector index tables stored in Elasticsearch, while the sentence graph is stored in Neo4j. The KNN graph connecting sentences is built using BM25-based similarity for fair comparison. At inference time, a query retrieves top-K sentences and generated memory via cosine similarity, expands sentence sets via graph traversal, and ranks parent chunks by aggregated sentence scores. Experiments used Qwen2.5-32B-Instruct as the LLM and evaluated on 500-question samples from LongMemEval and LoCoMo with LLM-as-a-Judge accuracy scoring.

Results: SGMem consistently outperformed all baselines on both benchmarks. SGMem-SMFI achieved 0.700 and 0.730 accuracy at Top-5 and Top-10 on LongMemEval, compared to the best RAG baseline RAG-SMFI at 0.676 and 0.680. On LoCoMo, SGMem-SF reached 0.522 and 0.542 versus 0.510 and 0.528 for RAG-SMFI. Graph-based RAG methods such as LightRAG, MiniRAG, and KG-Retriever underperformed due to misalignment with conversational granularity, while sentence-level graph expansion proved particularly effective for temporal reasoning and multi-session question types. Case studies confirmed that multi-hop traversal in SGMem enables retrieval of distant but relevant sessions that flat RAG misses.

TL;DR: SGMem organizes conversational memory as sentence-level graphs to overcome fragmentation and enable coherent retrieval across raw dialogue and generated memory, achieving state-of-the-art accuracy on long-term conversational QA benchmarks without requiring costly LLM-based extraction.

Beyond a Million Tokens: Benchmarking and Enhancing Long-Term Memory in LLMs

2026 Mohammad Tavakoli (University of Alberta), Alireza Salemi (University of Massachusetts Amherst), Carrie Ye (University of Alberta), Mohamed Abdalla (University of Alberta), Hamed Zamani (University of Massachusetts Amherst), J. Ross Mitchell (University of Alberta)
2510.27246v2.pdf Local PDF ↗ arXiv ↗
▸

This paper tackles the lack of realistic, large-scale benchmarks for evaluating long-term conversational memory in LLMs by introducing BEAM, a benchmark of 100 coherent dialogues up to 10M tokens with 2,000 probing questions across ten memory abilities. The authors also propose LIGHT, a cognitive-science-inspired framework combining episodic, working, and scratchpad memories to improve model performance. LIGHT achieves 3.5%–12.69% average improvement over the strongest baselines depending on the backbone model.

Problem being solved: Existing benchmarks for evaluating long-term memory in conversational LLMs have critical limitations: they construct long dialogues by artificially concatenating short sessions from different users, resulting in abrupt topic shifts and weak narrative coherence that simplify the evaluation. They also cover narrow domains (typically personal-life scenarios) and focus primarily on simple recall tasks, ignoring abilities such as contradiction resolution, event ordering, and sustained instruction following. As LLMs with context windows up to 1M tokens are deployed in multi-session conversational settings, there is an urgent need for benchmarks that test genuine long-range reasoning across diverse, coherent dialogues.

Approach: The paper introduces two complementary contributions. First, BEAM is a new benchmark built from a scalable generation pipeline that creates narratively coherent user-assistant conversations up to 10M tokens across diverse domains, accompanied by probing questions targeting ten distinct memory abilities. Second, LIGHT is a memory-enhancement framework inspired by human cognitive science that equips LLMs with three complementary memory systems: a long-term episodic memory index for retrieval, a short-term working memory of recent turns, and a scratchpad that accumulates salient facts incrementally throughout the conversation. At inference, the LLM draws on all three sources jointly to answer probing questions.

Methodology: BEAM conversations are generated in stages: a structured conversation plan with sub-plans is created from seed information including domain, user profile, and narrative arcs; user utterances are synthesized in batches from sub-plans; and assistant responses are generated with interactive follow-up and clarification loops. Probing questions are automatically produced targeting ten memory abilities and then human-validated, yielding 20 questions per conversation. Evaluation uses nugget-based scoring where atomic answer criteria are judged by an LLM, except for event ordering which uses Kendall tau-b. LIGHT indexes each dialogue turn as key-value pairs embedded in a vector database, maintains a scratchpad compressed when it exceeds 30K tokens, and filters the scratchpad at inference using semantic chunking and binary relevance labels.

Results: Experiments on BEAM show that all tested LLMs (GPT-4.1-nano, Gemini-2.0-flash, Qwen2.5-32B, Llama-4-Maverick) degrade substantially as conversation length grows from 100K to 10M tokens, with RAG providing limited relief at extreme lengths. LIGHT consistently outperforms both long-context and RAG baselines across all models and lengths, with gains of +49.1% for Llama-4-Maverick and +44.3% for GPT-4.1-nano at 100K tokens, rising to +155.7% and +107.3% respectively at 10M tokens. The largest improvements are in summarization (+160.6%), preference following (+76.5%), and temporal reasoning (+56.3%). An ablation study confirms that each component contributes increasingly as context length grows, with all four components essential at 10M tokens.

TL;DR: The authors present BEAM, a benchmark of multi-million-token coherent conversations, and LIGHT, a cognitively inspired three-memory framework, showing that standard LLMs fail at long-range conversational memory while LIGHT yields consistent and large improvements across all tested models and lengths.

LiCoMemory: Lightweight and Cognitive Agentic Memory for Efficient Long-Term Reasoning

2026 Zhengjun Huang (HKUST), Zhoujin Tian (HKUST), Qintian Guo (HKUST), Fangyuan Zhang (Huawei Hong Kong Research Center), Yingli Zhou (CUHK Shenzhen), Di Jiang (WeBank), Zeying Xie (WeBank), Xiaofang Zhou (HKUST)
2511.01448v2.pdf Local PDF ↗ arXiv ↗
▸

LLM agents lack persistent memory across long conversations due to finite context windows, and existing graph-based memory systems are heavy, redundant, and slow. LiCoMemory introduces CogniGraph, a lightweight hierarchical graph that acts as a semantic index rather than a knowledge repository, paired with temporally aware retrieval and reranking. The system achieves up to 23% accuracy improvement over the best baseline on long-term dialogue benchmarks while dramatically reducing update latency and token consumption.

Problem being solved: Large language model agents are constrained by finite context windows, meaning information from earlier conversations cannot be reliably recalled in long-term or multi-session interactions. Existing external memory approaches, particularly graph-based RAG systems, address this by storing conversational history in knowledge graphs, but they intertwine semantic content with relational topology in flat, entangled structures. This design leads to heavy and redundant representations, unstructured retrieval that returns fragmented or contextually inconsistent information, and prohibitively slow update and inference times. For example, systems like GraphRAG can require over 20 minutes for graph construction per dialogue and more than 2 minutes of query latency. These limitations make existing frameworks unsuitable for real-time, interactive agent deployment.

Approach: LiCoMemory proposes treating the knowledge graph as a lightweight semantic indexing layer rather than a content-heavy repository. The central innovation, CogniGraph, is a three-layer hierarchical graph comprising a session summary level, an entity-relation triple level, and a raw dialogue chunk level, connected by cross-layer hyperlinks. By storing only essential identifiers in graph nodes and edges and linking them back to original text, the graph remains compact and incrementally updatable. At inference time, the system performs top-down hierarchical retrieval guided by entity extraction from user queries, then reranks candidates using a unified score combining session-level similarity, triple-level similarity, and a Weibull-based temporal decay function. This design ensures that semantically relevant but temporally recent knowledge is prioritized without discarding older, still-relevant information.

Methodology: After each dialogue turn, the chunk with its timestamp and session ID is sent to LiCoMemory for incremental processing: session summaries are created or updated, entity-relation triples are extracted and deduplicated via type-aware and semantic similarity matching, and cross-layer hyperlinks are established. During retrieval, salient entities are extracted from the user query and matched against session summary keywords to identify relevant sessions, followed by entity-anchored traversal of the triple layer and mapping back to raw chunks. The unified reranking score is the product of the harmonic mean of session- and triple-level semantic similarities and a Weibull decay term that adapts its scale to the median temporal gap of retrieved triples. Experiments used Llama-3-8B-Instruct for memory construction, BGE-M3 for embeddings, and Llama-3.1-70B-Instruct-Turbo and GPT-4o-mini for answer generation, evaluated on LongMemEval (500 questions, 6 categories) and LoCoMo (1,986 questions, 5 categories).

Results: LiCoMemory consistently outperforms all baselines across both benchmarks and both generation models. On LongMemEval it surpasses the second-best baseline by 9.0% in accuracy and 9.3% in recall with Llama-3.1-70B; on LoCoMo it exceeds the next best by 7.5% accuracy and 5.2% recall. The largest gains appear in temporal reasoning (+19.2% on LoCoMo) and multi-session subsets (+26.6% on LongMemEval). In real-time interactive evaluation, LiCoMemory achieves 66.4% accuracy compared to 55.82% for the next best method, while reducing graph construction token cost by more than 3x and construction latency by over an order of magnitude (21 seconds versus 1,772 seconds for Mem0). Ablation studies confirm that structured hierarchical retrieval, temporal awareness, and session-level summaries each contribute substantially to overall performance.

TL;DR: LiCoMemory introduces CogniGraph, a three-layer hierarchical graph that separates semantic indexing from content storage, enabling real-time long-term memory for LLM agents that outperforms all baselines in accuracy and recall while being dramatically faster and more token-efficient.

MemoriesDB: A Temporal-Semantic-Relational Database for Long-Term Agent Memory

2025 Joel "val" Ward (CircleClick Labs, Austin TX)
2511.06179v1.pdf Local PDF ↗ arXiv ↗
▸

LLM agents suffer from context decoherence over long horizons because existing storage systems capture only one dimension of memory. MemoriesDB unifies time, semantic embeddings, and relational graph structure in a single append-only database built on PostgreSQL with pgvector, allowing agents to retrieve coherent, contextually grounded memories across extended timelines. A working prototype demonstrates scalable recall with sub-second interactive latency on datasets of tens of millions of records.

Problem being solved: Large language models lose coherence over extended interactions because their underlying storage mechanisms do not preserve the temporal, semantic, and relational structure of experience simultaneously. Sliding-window, RAG, and episodic-cache approaches mitigate token limits but do not provide a persistent substrate that encodes how knowledge evolves over time. Vector databases lack temporal ordering; time-series databases lack semantic representation; graph databases lack an embedded metric of similarity. Without a unified substrate, continuity must be reconstructed ad hoc from raw text, making long-horizon planning and self-referential reasoning unreliable.

Approach: MemoriesDB treats each memory as a first-class data entity that simultaneously encodes when an event occurred, what it means, and how it relates to other events — a triality of time, semantics, and relation. Every record is a vertex labeled by its microsecond timestamp, accompanied by normalized low- and high-dimensional embeddings, and connected to other vertices via labeled, weighted directed edges. The global structure forms a time-indexed stack of temporal-semantic surfaces, where edges project meaning across planes, enabling coherent traversal of long-horizon experience.
