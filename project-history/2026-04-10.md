### #261439 - Traces in Discover - Explore from APM - Transaction Detail - Unified waterfall size warning "view in Discover" link opens traces experience
- **Status**: Fresh
- **Created**: 2026-04-07
- **Fail Count**: 1
- **Diagnosis**: Flaky/Intermittent
- **Error**: `expect(locator).toBeVisible()` failed for `waterfallSizeWarning` with 10000ms timeout
- **Test File**: `src/platform/plugins/shared/discover/test/scout/ui/parallel_tests/traces_experience/explore_from_apm.spec.ts` (line 286)
- **Root Cause**: The test intercepts the `internal/apm/unified_traces/**` API route to force the `exceedsMax` condition, then navigates to the APM transaction detail page and expects the `waterfallSizeWarning` element to be visible. The route interception relies on timing -- the API response must be intercepted and modified before the UI renders. If the API call completes before the route handler is fully registered, or if the unified waterfall component takes longer to render in CI, the warning may not appear within the 10s default timeout. This feature was added in commit `08b052f1df10` (2026-03-25).
- **Action**: Increase the timeout for the `waterfallSizeWarning` visibility check, or add an explicit wait for the API route interception to be confirmed before navigating.
