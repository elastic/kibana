## Phase A — Harden on 5.9.3

### A1 — `downlevelIteration` removed

No-op at `target: "esnext"`, deprecated in TS 6.0, removed in TS 7.0.

### A2 — Sync base configs (reframed, no code change)

`tsconfig.base.type_check.json` is auto-generated by `createTypeCheckConfigs` from `tsconfig.base.json` whenever a project with `extends: @kbn/tsconfig-base/tsconfig.json` is built. Scoped runs only regenerate the targeted projects; full runs regenerate the base. No separate sync mechanism needed.

### A3 — `moduleResolution: "node"` → `"bundler"` (deferred, blocking TS 7.0)

Paired change required: `module: "commonjs"` → `"preserve"`. Probe on `src/core/packages/plugins/server-internal/tsconfig.json` after the flip:

- 121 errors, top categories:
  - 43 × TS2305 — `@elastic/opentelemetry-node/sdk` subpath types not visible under bundler resolution (the ambient typings in `typings/@elastic/opentelemetry-node/sdk` only get picked up by `node10` classic resolution; bundler looks at the package `exports` map).
  - 6 × TS1192 — `import api from '@elastic/elasticsearch/lib/api/types'` (namespace, not default).
  - 1 × TS2307 — deep `@opentelemetry/api/build/src/metrics/Metric` import.
  - 6 × TS2578 — newly-unused `@ts-expect-error` (cleanup win).
  - 37 × TS2322 / 13 × TS2344 / 3 × TS2339 — downstream cascade.

Sequencing (each a small PR, all on `moduleResolution: "node"`):
1. Fix `@elastic/elasticsearch` namespace imports.
2. Fix the `@opentelemetry/api` deep import.
3. Reshape `typings/@elastic/opentelemetry-node/sdk` as a proper module with `exports`, or move to a shim package.
4. Audit and remove unused `@ts-expect-error` lines surfaced (TS2578).
5. Flip the base flag (`"moduleResolution": "bundler"` + `"module": "preserve"`).

### A4 — `@ts-expect-error upgrade typescript v5.9.3` audit (folded into C2)

Hundreds of occurrences. Phase C bump will surface the unused ones as TS2578; delete in C2 fallout. Anything still required gets retagged to `v6.0.3`.

### A5 — `@typescript-eslint` bump

Done in Phase C: `8.46.3` (peer `>=4.8.4 <6.0.0`) → `8.60.0` (peer `>=4.8.4 <6.1.0`).

## Phase C — Default to TS 6.0.3

  - `"typescript": "6.0.3"` (was `5.9.3`); alias removed.
  - `"**/typescript": "6.0.3"` resolution.
  - `"@typescript-eslint/{eslint-plugin,parser,typescript-estree}": "8.60.0"` (was `8.46.3`).
  - `"**/@typescript-eslint/utils": "8.60.0"` resolution.
- `tsconfig.base.json`: added `"ignoreDeprecations": "6.0"`.
- `tsconfig.base.type_check.json` (auto-generated): same.
- Reverted Phase B scaffolding (`KBN_TS_COMPILER_PACKAGE`, `resolve_tsc_binary.ts`, `check_types_ts6.sh`, the soft-fail Buildkite step).

### Smoke results

- `node scripts/type_check --project packages/kbn-ts-type-check-cli/tsconfig.json` → green in 3.1s.
- `node scripts/type_check --project src/core/packages/plugins/server-internal/tsconfig.json` → green in 39.7s.
- `node scripts/check_changes.ts` → green.
- ESLint smoke on the modified CLI files → green.

### Risks / things to watch on the first CI run

- **GCS type-check cache invalidates entirely** because `yarn.lock` changed (see `CACHE_INVALIDATION_FILES` in `packages/kbn-ts-type-check-cli/src/archive/constants.ts`). First CI run is going to be slow.
- **Jest / Babel / SWC compile paths** were not exercised locally. Babel `@babel/preset-typescript@7.24.7` strips types only and TS 6 ships no new syntax, so it should be a non-event. Cypress and Scout share the same Babel pipeline.
- Some transitive dev-deps still pin `typescript@<6.0.0` in their peers (`find-cypress-specs`, etc.). They print warnings; not blocking.

## Phase C2 — Full-repo typecheck fallout (2026-05-29)

First full `node scripts/type_check` on TS 6.0.3 surfaced **56 errors**. Saved to `typecheck.log.txt` (untracked).

### What landed

3. **C2 inference adapters** (`32522046fb80`): explicit `OperatorFunction` return types on `processOpenAIStream` / `processOpenAIResponse`; dropped the generic param on `emitTokenCountEstimateIfMissing`; pinned the `identity`/`parseInlineFunctionCalls` ternary fallback (TS2345 + TS2322).
4. **C2 ai-assistant mocks** (`a0850d0dfae8`): cast over-typed `mockResolvedValueOnce` / `mockImplementation` through unknown to `Parameters<...>[0]` in `use_conversation.test.tsx` (TS2345 × 3).
