# i18n in @kbn/one-navigation

The source navigation component uses `@kbn/i18n` and `@kbn/i18n-react` for
internal strings (e.g. the "More" overflow label and screen-reader
instructions). Consumer-facing labels like "Dashboards" or "Settings" are
provided through the navigation items data structure and do not go through i18n.

## How it works

At build time, webpack aliases redirect all `@kbn/i18n*` imports to a no-op
stub (`packaging/react/services/i18n.tsx`). The stub's `translate()` function
simply returns `defaultMessage`, so internal strings render in English.

If your application requires translated internal strings, replace the stub with
your own implementation that conforms to the same API shape.
