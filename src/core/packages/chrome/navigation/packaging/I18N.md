# Internationalization

## Current Status

**Your labels**: Any language (you provide via props)  
**Internal strings**: English only

## Your Navigation Labels

You control all navigation item labels. Use your product's i18n solution:

```tsx
import { useTranslation } from 'react-i18next';

function MyApp() {
  const { t } = useTranslation();
  
  const items = {
    primaryItems: [
      { id: 'dashboard', label: t('nav.dashboard'), iconType: 'dashboardApp', href: '#/' },
      { id: 'analytics', label: t('nav.analytics'), iconType: 'graphApp', href: '#/analytics' },
    ],
  };
  
  return <OneNavigation items={items} {...props} />;
}
```

## Internal Strings (English Only)

Component-generated strings remain in English:

| String | When Visible | Impact |
|--------|--------------|--------|
| "More" | Overflow menu (many items) | Medium - visible to users |
| "Skip to main content" | Always (screen readers) | Low - standard term |
| ARIA labels | Always (screen readers) | Low - standard terms |

**Mitigation**: Keep navigation items to 7-10 max to avoid overflow menu.

## FAQ

**Q: Can I use this in non-English products?**  
A: Yes. You control all visible labels. Only internal strings (like overflow "More") remain in English.

**Q: Should I wait for multi-language support?**  
A: No, unless you require every string translated. Current implementation is production-ready.

**Q: How do I request multi-language support?**  
A: Contact Shared UX via #appex-sharedux or GitHub Issues with your use case and timeline.
