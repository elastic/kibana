## Approach and Hypothesis

The test `<HighlighedFieldsModal /> should update rule when save button is clicked` was intermittently timing out (5000ms Jest default timeout) due to a React Testing Library `act()` tracking issue. The failing test used the raw DOM `.click()` method (`getByTestId(...).click()`) inside an `await act(async () => { ... })` block, while the other test in the same file correctly used `fireEvent.click()`. In React 18 concurrent mode, firing events via the native DOM `.click()` API can cause `act()` to not properly track the async work generated by the event handler. Since `onSubmit` is an async function (it `await`s `updateRule()`), `act()` needs to properly detect and wait for the resulting microtask continuation and any React state updates. When using native `.click()`, React's internal async work tracking can miss the pending microtasks from the async handler, causing `act()` to either return prematurely (flaky assertion) or — as observed — hang waiting for work it can't see (timeout). Replacing `.click()` with `fireEvent.click()` ensures the event is fired through RTL's event system, which integrates correctly with React's `act()` batching and async tracking.
