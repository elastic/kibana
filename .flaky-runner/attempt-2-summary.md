## Approach and Hypothesis

Attempt 1 changed `.click()` to `fireEvent.click()` but kept the `await act(async () => { ... })` wrapper — this still failed because the root cause was the `act()` call itself hanging, not the click method. The real culprit is the `useField` hook inside Kibana's `hook_form_lib`, which sets a `valueChangeDebounceTime` of 500ms by default via `setTimeout`. The component's form schema uses `fieldsToValidateOnChange: ['customHighlightedFields']`, so when the `CommonUseField` (combo box) mounts, the field registers a debounced validation timer. In React 18's test environment, `act()` can detect this pending `setTimeout`-backed timer as "pending React work" and wait indefinitely for it to complete, causing the 5000ms test timeout. The fix removes the `act()` wrapper entirely and replaces it with `fireEvent.click()` followed by `await waitFor(() => expect(mockSetIsModalVisible).toHaveBeenCalledWith(false))`. `waitFor` polls the assertion using retries without trying to drain React's entire async work queue (including the pending debounce timer), so the test completes in ~50ms as soon as the async `onSubmit` callback resolves and calls `setIsModalVisible(false)`.
