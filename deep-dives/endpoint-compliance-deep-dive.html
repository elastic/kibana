Overview

Endpoint Compliance Monitoring extends Kibana's Osquery plugin to evaluate managed endpoints against CIS benchmarks (macOS 15, Windows 11, Linux RHEL 9/Ubuntu 22.04) and NIST 800-53 controls. It ships 30 prebuilt compliance rules as osquery SQL queries, evaluates results into pass/fail/N/A findings, aggregates posture scores per benchmark, and renders a Security Solution–style dashboard.

The feature is gated behind the endpointComplianceMonitoring experimental flag, uses internal-only API routes, and aligns its data model with Cloud Security Posture (CSP) for future integration. It supports multi-vendor endpoints via a strategy pattern (Elastic Defend, SentinelOne, Microsoft Defender).

  • 30 prebuilt CIS rules across 3 platforms with NIST 800-53 r5 framework mappings
  • Real-time evaluation polling osquery results every 30 seconds
  • Score aggregation via Task Manager on a 5-minute interval
  • 12 versioned API routes with RBAC authorization
  • 3 UI pages: Dashboard, Findings Explorer, Rules Management
  • Detection rule bridge generating SIEM threshold rules with MITRE ATT&CK mapping

How It Works

Step 1: Rule Definition

Each compliance rule is a ComplianceRuleMetadata saved object containing an osquery SQL query, benchmark metadata (CIS macOS 15, CIS Windows 11, CIS Linux), a section (e.g. “2 Network”), level (1 or 2), platform, and NIST 800-53 framework control mappings. 30 rules ship prebuilt; custom rules can be created via API.

Step 2: Pack Deployment

packDeploymentService.buildCompliancePack() reads all enabled rules for a benchmark, constructs an osquery pack config with schedule IDs prefixed compliance-, and pushes it to Fleet agent policies via deployCompliancePack(). Elastic Agents with the osquery integration then execute these queries on managed endpoints.

Step 3: Osquery Execution

The Elastic Agent runs osquery SQL queries on each endpoint per the scheduled interval (default 300s). Results flow into logs-osquery_manager.result-* in Elasticsearch.

Step 4: Finding Evaluation

The FindingEvaluatorService polls logs-osquery_manager.result-* every 30 seconds for results matching the compliance- schedule ID prefix. For each result, it evaluates:

  • Rows returned → passed
  • No rows → failed
  • Query error → not_applicable

Findings are bulk-written to logs-endpoint_compliance.findings-default.

Step 5: Score Aggregation

A Task Manager task runs computeAndWriteScores() every 5 minutes. It aggregates findings per benchmark (excluding muted rules), calculates score = round(passed / (passed + failed) * 100, 1), and writes snapshots to the scores data stream. The continuous transform deduplicates findings into findings_latest.

Step 6: Dashboard Rendering

The React dashboard calls GET /internal/endpoint_compliance/stats/{benchmark_id}, which runs a complex aggregation returning section scores (worst 5), host scores (worst 10), total counts, and the score trend. The UI renders the score gauge, trend chart, section table, and worst hosts table.

Configuration & Setup

Enable the Feature Flag

# config/kibana.dev.yml
xpack.osquery.enableExperimental:
  - endpointComplianceMonitoring

Prerequisites

  1. Elasticsearch running (the plugin creates index templates and ILM policies on startup)
  2. Osquery Manager integration installed via Fleet
  3. At least one osquery package policy added to an agent policy

Seed Sample Data

# From the osquery plugin directory:
bash scripts/seed_compliance_data.sh \
  --es-url http://localhost:9200 \
  --hosts 10 --days 7

Run Kibana

yarn start --no-base-path

Navigate to Osquery → Compliance in the Kibana sidebar.

UI Pages & Components

Dashboard

Benchmark selector cards → score gauge (color-coded at 80/60% thresholds) → score trend line chart → worst 5 CIS sections → worst 10 hosts. Time range: 24h / 7d / 30d.

Findings Explorer

Filterable table (benchmark, evaluation, search) with paginated results. Columns: Result (color badge), Rule, Section, Level, Host, OS, Timestamp. Row click opens a detail flyout with remediation steps and evidence JSON.

Rules Management

30 rules with checkbox selection for bulk actions (enable/disable/mute/unmute). Columns: #, Name, OS (platform icon), Level, Section, Frameworks (NIST badges), Status, Source. Row click opens flyout with osquery SQL query, remediation, framework mappings, and interval.
