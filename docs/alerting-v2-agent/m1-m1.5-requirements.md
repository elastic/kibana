## Cross-Cutting Issues to Resolve Before M1.5

### Audit Log

Requirements S4, A5, and the Migration governance requirement all describe agent action logging. These should resolve to **one unified audit mechanism** rather than three independent implementations.

- **Proposed decision:** Define a single `agent_action_log` structure at M1 that all flows write to. Confirm at M1.5 that authoring bulk edits use it. Migration governance can reuse at GA.

### Simulation Infrastructure

Requirements A3 (simulation assistant), the Debugger re-test loop (M2/GA), and Migration validation (GA) all depend on historical window replay. This is a shared infrastructure capability with no dedicated requirement row.

- **Proposed decision:** Add an explicit cross-cutting requirement for a "simulation/replay engine" and assign it to M1.5 since A3 blocks on it.

### RBAC Error UX

S1 requires RBAC enforcement but no requirement describes what users see when an action is blocked. Without this, flows will have undefined behavior at permission boundaries.

- **Proposed decision:** Add a requirement at M1 for a standardized agent permission-denied response pattern (message + reason + suggested alternative action).
